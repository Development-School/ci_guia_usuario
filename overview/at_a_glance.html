Visão Geral do CodeIgniter

O CodeIgniter é uma Aplicação Frameworks

CodeIgniter é um kit de ferramentas para quem constrói aplicações web usando PHP. Seu objetivo é que você possa desenvolver projetos muito mais rápido do que você poderia se você estivesse escrevendo código a partir do zero, proporcionando um rico conjunto de bibliotecas para as tarefas mais comuns, bem como uma interface simples e estrutura lógica para acessar essas bibliotecas. CodeIgniter permite que você mantenha o foco em seu projeto minimizando a quantidade de código necessário para realizar uma tarefa.

O CodeIgniter é gratuito

O CodeIgniter é disponibilizado sob uma licença de código aberto no estilo da Apache/BSD. Sendo assim, você pode utilizá-lo como desejar. Para mais informações, por favor leia a Licença de Uso.

CodeIgniter é muito leve

Verdadeiramente levíssimo. O núcleo do sistema requer apenas algumas pequeníssimas bibliotecas. Isto é um contraste gritante com muitos frameworks que requerem significativamente mais recursos. Bibliotecas adicionais são carregadas dinamicamente a pedido, com base em necessidades de um determinado processo do seu aplicativo, de modo que o sistema básico é enxuto e bem rápido.

CodeIgniter é rápido

Extremamente rápido. Nós o desafiamos a encontrar um framework que tem melhor desempenho do que o CodeIgniter.

O CodeIgniter Usa M-V-C

O CodeIgniter usa a abordagem Modelo-Visão-Controlador, que permite grande separação entre lógica e apresentação. Esta particularidade é boa para projetos que são desenhados para trabalhar com arquivos de modelos, como os códigos que aqueles arquivos contém serão minimizados. Nós descrevemos o MVC em mais detalhes em sua própria página.

O CodeIgniter Gera URLs Limpas

As URLs geradas pelo CodeIgniter são limpas e amigáveis aos motores de busca. No lugar de usar a abordagem padrão "query string" para as URLs que é sinônimo de sistemas dinâmicos, CodeIgniter usa uma abordagem baseada em segmentos:

exemplo.com/noticias/artigo/345

Nota: Por padrão o arquivo index.php é incluído na URL mas ele pode ser removido usando um simples arquivo .htaaccess.

O CodeIgniter vem Recheado de Coisas

O CodeIgniter vem com uma grande variedade de bibliotecas que abrange a maioria das tarefas do desenvolvimento web, como acesso a um banco de dados, mandar e-mail, validar dados de um formulário, manter sessões, manipular imagens, trabalhar com dados XML-RPC e muito mais.

O CodeIgniter é Extensível

O sistema pode ser facilmente estendido através do uso de suas próprias bibliotecas, Helpers, ou através de classe extensões ou ganchos de sistema.

O CodeIgniter Não requer um Gerador de Templates

Embora o CodeIgniter venha com um simples interpretador de templates, opcional, ele não lhe força a usar um. Geradores de template simplesmente não alcançam a performance do PHP nativo, e a sintaxe que precisar ser aprendida para usá-los é geralmente um pouco mais fácil que aprender os básicos do PHP. Considere este bloco de código PHP:

<ul>

<?php foreach ($addressbook as $name):?>

<li><?=$name?></li>

<?php endforeach; ?>

</ul>

Compare com este pseudo-código usado por um gerador de templates:

<ul>

{foreach from=$addressbook item="name"}

<li>{$name}</li>

{/foreach}

</ul>

Sim, o segundo exemplo é um pouco mais claro, mas ele cobra seu preço na performance, pois um pseudo-código precisa ser convertido novamente em PHP pra rodar. Como um de nossos objetiso é performance máxima, optamos por não requerer o uso de um gerador de templates.

O CodeIgniter é Exaustivamente Documentedo

Programadores adoram codificar e odeiam documentar o que fazem. Nós não somos diferentes, é claro, mas como a documentação é tão importante que o código propriamente dito, somos dedicados em fazê-lo. Nosso código fonte é extremamente limpo e também bem documentado.

O CodeIgniter tem uma Comunidade de Usuários Amigável

Nossa crescente comunidade de usuários pode ser vista participando ativamente em nosso Fórum da Comunidade.
